Ordered Factors and Reordered Tables (cont.)

Remember, from the Arthritis Data Example, we re-orderd the levels of the variable Improved. Factor variables in data frames (case form or frequency form) can be re-ordered and declared as ordered factors using ordered().

The order of the factors themselves can be rearranged by sorting the data frame using sort().

Currently, the way JobSat is defined in table form, the values of the table factors are ordered by their position in the table.

What if we need the levels of income and satisfaction to be numeric variables?

Generating tables with table() and xtabs()

When the data is in case or frequency form, you can generate frequency tables from factor variables in data frames using the table() function.

  • for tables of proportions, use prop.table()
  • for marginal frequenices (summing over variables), use margin.table()

table()

table() (cont.)

The table() functions takes a lists of variables as it’s arguments. The variables are factors. There is no data= argument. So, you must specify the names of columns in the data frame, or select the variables using column indexes.

# 2-way Frequency Table

table(mydata$A, mydata$B) #A will be the rows, B will be the columns
##     
##      b1 b2
##   a1 25 21
##   a2 22 32
mytab <- table(mydata[,1:2])   #produces same result
mytab
##     B
## A    b1 b2
##   a1 25 21
##   a2 22 32

margin.table(X, margin)

Use margin.table(X, margin) to sum a table X for the indices in margin, that is, over the dimensions not included in margin.

margin.table(mytab) # sum over A & B
## [1] 100
margin.table(mytab, 1) # A frequencies (summed over B)
## A
## a1 a2 
## 46 54
margin.table(mytab, 2) # B frequencies (summed over A)
## B
## b1 b2 
## 47 53
addmargins(mytab) # show all marginal totals
##      B
## A      b1  b2 Sum
##   a1   25  21  46
##   a2   22  32  54
##   Sum  47  53 100

prop.table()

The function prop.table() expresses the table entries as a fraction of a given marginal table.

prop.table(mytab) # cell percentages
##     B
## A      b1   b2
##   a1 0.25 0.21
##   a2 0.22 0.32
prop.table(mytab, 1) # row percentages
##     B
## A           b1        b2
##   a1 0.5434783 0.4565217
##   a2 0.4074074 0.5925926
prop.table(mytab, 2) # column percentages
##     B
## A           b1        b2
##   a1 0.5319149 0.3962264
##   a2 0.4680851 0.6037736

table() (cont.)

xtabs()

The xtabs() function allows you to create cross tabulations of data using formula style input. This typically works with case-form or frequency-form data supplied in a data frame or a matrix. The result is a contingency table in array format, whose dimensions are determined by the terms on the right side of the formula.

# 3-Way Frequency Table 
mytable <- xtabs(~A+B+sex, data=mydata) 
mytable
## , , sex = F
## 
##     B
## A    b1 b2
##   a1 14  8
##   a2  7 18
## 
## , , sex = M
## 
##     B
## A    b1 b2
##   a1 11 13
##   a2 15 14
ftable(mytable) # print table
##       sex  F  M
## A  B           
## a1 b1     14 11
##    b2      8 13
## a2 b1      7 15
##    b2     18 14

Printing tables with structable() and ftable()

For 3-way and larger tables, the functions ftable() (in the stats package) and structable() (in vcd) provide a convenient and flexible tabular display in a ‘flat’ (2-way) format.

With ftable(X, row.vars=, col.vars=), variables assigned to the rows and/or columns of the result can be specified as the integer numbers or character names of the variables in the array X.

By default, the last variable is used for the columns.

The formula method, in the form ftable(colvars ~ rowvars, data) allows a formula where the left- and right-hand side of formula specify the column and row variables, respectively.

Subsetting Data

Many times you only want to examine a subset of data, for example, you only want to analyze students with only Brown Hair and Blue Eyes. Or, you only want to focus on students that were Admitted to Berkley University.

This section is going to show us how to subset data in tables, structables and data frames.

Subsetting Tables

If data are available in tabular form created with table() or xtabs(), resulting in table objects, subsetting is done via indexing, either with integers or character strings corresponding to the factor levels.

The following code extracts the female data from the HairEyeColor data set:

#same as HairEyeColor[,,2]
#empty indices means include all that data
#output is reduced to a two-way table
HairEyeColor[,,"Female"]

##   Brown    66   34    29    14

HairEyeColor[c("Black", "Brown"), c("Hazel", "Green"),]

Subsetting Tables (cont.)

Functions like apply() can iterate through all levels of one or several dimensions and apply a function to each subset. The following calculates the total amount of male and female students:

apply(HairEyeColor,3,sum)

Collapsing Tables

Collapsing Over Table Factors; aggregate(), margin.table(), and apply()

Many times data sets will have more variables or factors than we want to analyze or visualize. So, we can exclude the variables or factors by collapsing (that is, summing) over the variable or factor.

The appropriate R function to perform the summing depends on how the data are represented.

aggregate() is appropriate when the data is in

  • case form data frame
  • frequency form data frame

margin.table() or apply() are appropriate when the data is in

  • array
  • table

Collapsing Table Levels collapse.table()

For tables or arrays, you may want to reduce the levels (categories) of a factor by summing some of the subsets of the frequencies.

Example 2.8

Let’s generate some data and collapse age from 10-year to 20-year intervals and education from three levels to two.

# create some sample data in frequency form
sex <- c("Male", "Female")
age <- c("10-19", "20-29", "30-39", "40-49", "50-59", "60-69")
education <- c("low", 'med', 'high')
dat <- expand.grid(sex=sex, age=age, education=education)
counts <- rpois(36, 100) # random Poisson cell frequencies
dat <- cbind(dat, counts)
#creating 3-way table
tab1 <- xtabs(counts ~ sex + age + education, data=dat)
structable(tab1)
